Step 1

Initial Agreement

Before opening the dashboard, agree on all terms with the consignor. This avoids needing to edit the record later.

Field Decision needed Notes
Item One specific item per consignor record e.g. "Wetware Mainframe". Multiple items = multiple records.
Slot Type Exclusive or Shared Exclusive — only they supply this item. Shared — another consignor also covers it.
Commission % Their cut of each sale e.g. 70 = they keep 70%, you keep 30%.
Max Units Stock cap you'll accept at once Prevents overstocking. Set based on monthly sell velocity.
Demand Tier High / Medium / Low Your assessment of how fast this item sells in J-space.
List Price Price per unit Can be left blank and filled in later once you've checked Jita.
Important

For shared slots, also agree on Priority: who gets sales attributed first when both have stock. Priority 1 = primary supplier; Priority 2 = secondary/filler.

Step 2

Add the Consignor in the Dashboard

Open Admin Dashboard → Consignment tab.

  1. Click + Add Consignor
  2. Fill in the dialog fields:
    • Character Name — exact in-game name
    • Item Name — the item they supply
    • Item Type ID — the number from EVE's market (see reference below). Required for contract sync.
    • List Price / Unit — agreed ISK per unit
    • Their Commission % — e.g. 70
    • Slot Type — Exclusive or Shared
    • Priority — 1 for primary, 2 for secondary
    • Max Units — stock cap
    • Demand Tier — High / Medium / Low
    • Start Date — today's date (auto-filled)
  3. Click Save

The consignor appears in the Consignors treeview. Status shows Active.

Finding the Item Type ID

In-game: right-click the item → Show Info. The type ID is displayed in the info window title bar.
EVE Marketer: go to evemarketer.com, search the item — the type ID is in the URL (typeID=XXXXX).
Janice / evepraisal: paste the item name; the type ID appears in the JSON output.

Step 3

Set Up Their Container In-Game

Each consignor gets a dedicated hangar container. This is what makes inventory tracking automatic — the system reads container contents every sync and updates the consignor's In Slot quantity.

  1. In EVE, open your personal hangar at LX-ZOJ
  2. Place a container there — any type works (Giant Secure Container recommended for capacity)
  3. Name it clearly — right-click → Rename
    Suggested format: [CharName] — [Item]
    Example: Hamektok — Wetware Mainframe
    ESI reads this name and stores it in the dashboard automatically.
  4. Give the consignor access to deliver to you, or take delivery yourself via contract
Finding the Container Item IDMethod
In-game Right-click container → Show Info. Note the Item ID shown in the info window.
After first sync The container will appear in the Container Assignments treeview once assigned and synced. The Item ID column shows the number.
ESI asset viewer Tools like Evernus or Adam4EVE can browse your assets and show item_ids.
Critical Rule

Keep the consignor's container at the top level of your personal hangar — do not nest it inside another container. The tracking system uses the top-level container ID as the assignment key. Nesting changes how IDs are resolved.

Container Stability

The container's Item ID is permanent as long as the container exists. If the container is destroyed or replaced with a new one, you'll need to update the assignment in the dashboard with the new ID.

Step 4

Assign the Container in the Dashboard

In the Consignment tab, scroll down to the Container Assignments pane.

  1. Click + Assign Container
  2. Select the consignor from the dropdown
  3. Enter the Container Item ID from Step 3
  4. Add a note if helpful (e.g. Row 2, Giant Secure Container)
  5. Click Save

The assignment appears in the table. The ESI Name column shows (no name yet) until the first sync.

One Container per Consignor

Each container assignment is one-to-one: one container → one consignor record. If a consignor supplies two different items, create two consignor records (one per item) and assign a separate container to each.

Step 5

Run the First Inventory Sync

In the Container Assignments pane, click ↺ Refresh Names. This triggers a full ESI inventory sync in the background.

The sync does the following in one pass:

  1. Fetches all personal assets from ESI for the LX-ZOJ hangar
  2. Walks container contents using ESI's asset graph
  3. Reads custom container names from ESI (/assets/names/)
  4. Updates container_name in the Container Assignments table
  5. Stores a per-container inventory snapshot in the database
  6. Updates each consignor's In Slot quantity based on what's in their assigned container
  7. Regenerates the live site inventory display and pushes to GitHub Pages

When complete, a dialog confirms success. The ESI Name column will now show the container's in-game name, and the Consignors treeview will show the consignor's current stock quantity.

Automatic Schedule

After the initial setup, inventory syncs run automatically every 2 hours via the Windows Task Scheduler. Manual syncs are only needed for immediate updates or initial onboarding.

Step 6

Notify the Consignor

Once setup is complete, send the consignor the following information. A suggested message template:

Hey [name], your consignment slot is set up! Here are your details:

  • Item: [item name]
  • Your container: [container name] (in my personal hangar at LX-ZOJ)
  • Max stock at one time: [max units] units
  • List price: [price] ISK/unit
  • Your commission: [their %]% per unit sold
  • Delivery: contract to me in-game, or drop directly if you have hangar access

I'll pay out after each sales cycle. Contact me anytime if you have questions. o7

Tip

Point them to the live site — the Inventory tab shows real-time stock levels so they can see when their container is running low and schedule a restock.

Ongoing

Sales, Attribution & Payouts

When the consignor restocks

Nothing to do. The next scheduled sync will detect the new units in their container and update In Slot automatically. You can also click ↺ Refresh Names for an immediate update.

Syncing contract sales — Exclusive slot

  1. Ensure fetch_contract_profits.py has run (or wait for its schedule)
  2. In the Sales Log toolbar, click ↺ Sync Contracts
  3. Sales are auto-logged with value-weighted price attribution
  4. The ISK Owed column in the Consignors treeview updates immediately

Syncing contract sales — Shared slot

  1. Sync Contracts as above
  2. Shared-slot sales appear in the Pending Attribution pane (orange) instead of auto-logging, because the system cannot determine which consignor's units were sold
  3. Select the pending row → click Split / Attribute
  4. The dialog shows all active consignors for that item with their current stock
  5. Click Auto-fill (FIFO by priority) to pre-fill quantities based on priority order and current stock, or enter quantities manually
  6. Click Save Attribution — individual sale entries are logged per consignor and their In Slot quantities are decremented

Paying out

  1. Select the consignor in the Consignors treeview — their Sales Log loads below
  2. Review unpaid entries (shown in gold)
  3. Transfer ISK in-game
  4. Back in the dashboard: select all unpaid rows → Mark Selected Paid
Payout Cadence

There is no enforced schedule — set your own (weekly is common). Be consistent and communicate it to consignors upfront to avoid disputes.

Offboard

Offboarding a Consignor

When a consignor stops supplying or the arrangement ends:

  1. Return any remaining stock from their container via contract or direct trade
  2. Settle any outstanding ISK owed — mark all entries Paid
  3. In the Consignors treeview → select their row → Toggle Active
    This marks them Inactive and greys out the row. Historical sale records are preserved.
  4. In Container Assignments → select their row → Remove
    This removes the tracking assignment only — the physical container in-game is unaffected.
  5. In EVE: empty the container, rename or remove it as preferred
Record Preservation

Deactivating a consignor does not delete their sale history. Records remain in the database for accounting purposes. If they return later, simply re-activate via Toggle Active and reassign a container.

Troubleshooting

ProblemLikely causeFix
In Slot stays 0 after sync Container ID mismatch or container not assigned yet Verify the Container Item ID in the assignment matches the actual in-game container. Reassign if needed.
ESI Name still shows "(no name yet)" Container not named in-game, or sync hasn't run since naming Name the container in-game, then click ↺ Refresh Names.
Contract sync finds nothing Item Type ID missing or wrong on consignor record Edit the consignor → check/correct the Item Type ID. Also ensure fetch_contract_profits.py has run recently.
Sale goes to Pending instead of auto-logging Two active consignors share the same Item Type ID This is correct for shared slots. Use Split / Attribute to resolve.
Consignor missing from Split dialog Their record is Inactive or has no Item Type ID set Edit the consignor → set Item Type ID. Toggle Active if they were deactivated.
Refresh Names shows "ESI refresh failed" Auth token expired or ESI is down Re-authenticate via token_manager.py, then retry. Check ESI status at status.eveonline.com.
current_qty shows wrong number Items in container not in tracked_market_items The container may hold other untracked items; only tracked type_ids count. Check if the item's type_id is in the tracked list.
